Product Requirement Document (PRD)Product Name: EcoVoice (Temporary Working Title)Author: Anirudh JainTarget Platform: Native Desktop (macOS / Windows / Linux via Tauri or Electron)Core Value Proposition: A 100% private, serverless, system-wide speech-to-text writing assistant that converts voice to text locally and instantly polishes grammar/phrasing for non-native English speakers on demand.1. Executive Summary & ObjectiveFor many professionals and developers, English is a second language. While standard voice-to-text tools work well, they transcribe raw speech verbatim—complete with broken grammar, filler words, and awkward sentence structures.EcoVoice solves this by providing a global desktop shortcut that acts as an invisible typing assistant. By default, it inputs fast, local speech-to-text. With a single click or modifier key, it utilizes an ultra-small, open-source local LLM to restructure and polish the text into natural, professional English before injecting it into the active text field.Because everything runs locally via optimized native runtimes, the app requires zero server infrastructure, guarantees absolute privacy, and incurs zero continuous API costs for the developer or user.2. Target Audience & PersonasThe Global Professional: Software engineers, creators, and managers who speak English as a second language and want to communicate smoothly in Slack, emails, or documentation without second-guessing their grammar.The Privacy-First User: Enterprise employees who handle sensitive data or proprietary code and are legally blocked from streaming audio or text logs to cloud APIs (like OpenAI or cloud-based dictation tools).3. The Component FrameworkFrontend/Desktop Shell: Tauri (Rust + TS) or Electron. Recommendation: Tauri keeps the binary size incredibly tiny (under 10-15MB without model weights) and consumes significantly less RAM than Electron.Speech-To-Text (ASR): whisper-node wrapper around whisper.cpp running the whisper-base or whisper-small model.Text Refinement (LLM): node-llama-cpp natively orchestration running Llama-3.2-1B-Instruct or Qwen-2.5-1.5B-Instruct quantized to 4-bit (Q4_K_M GGUF format).4. Key Features & User WorkflowsFeature 1: The "Hold-to-Talk" System DictationDescription: The user holds a global hotkey (e.g., Option + Space or a mouse click) and talks into their microphone. A minimal floating overlay displays an active audio waveform indicator.Workflow:User holds shortcut key combination.Microphone stream initiates, writing raw audio frames directly into memory.User releases shortcut key combination.Audio buffers are flushed to whisper.cpp.Raw text is computed locally.Feature 2: Dual Mode Execution (Direct vs. Polish)Mode A (Default Fast Entry): The raw text is instantly injected directly into the user's cursor position in the active application (e.g., VS Code, Slack, Browser).Mode B (Improve Writing / Accent Polish): If the user toggles the "Improve" workflow (e.g., holding a secondary key like Shift while releasing the mic hotkey, or clicking a floating floating toolbar indicator), the text processing deviates:Raw Whisper transcript is caught in memory.The string is fed directly into the local 1B LLM context via a deterministic context template.The model corrects grammar gaps, adjusts phrasing to standard English idioms, and strips verbal pauses.The polished output text is injected instead of the raw text.Feature 3: On-Device Model Asset ManagementDescription: To keep the initial desktop application downloader small, the base application bundle contains no weights.Workflow: On first boot, a setup wizard guides the user through downloading their preferred language files directly to their local application data storage folders (Appdata or Application Support).Whisper Base Model: ~140 MBLlama 3.2 1B (Quantized): ~1.2 GB5. Phase 1 MVP Scope (The Blueprint)To validate the product quickly, Phase 1 will completely bypass building complex setting sliders, multiple account syncs, or custom themes.
